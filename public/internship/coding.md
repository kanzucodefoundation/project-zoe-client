# Project Zoe Internship — Coding Track

## What it is
Small, well-scoped contributions to the Project Zoe server (NestJS/Prisma) or client (React), tagged and ready to pick up. Sprint length: 4-6 weeks.

## Entry bar
- Comfortable with JavaScript/TypeScript basics and git
- Can get both repos running locally from the README ([server](https://github.com/kanzucodefoundation/project-zoe-server), [client](https://github.com/kanzucodefoundation/project-zoe-client))

## Trial task (this is the real interview)
Within 48 hours, unsupervised:
1. Clone both repos, get the client talking to the server locally
2. Pick one issue tagged `good-first-issue`
3. Open a PR against `develop` that satisfies the issue's acceptance criteria

If you complete this well, you're in — no separate interview.

## How work is assigned
- All tasks live as GitHub issues tagged `good-first-issue`, each with a clear "done" definition
- One technical mentor per 2-3 interns, two short pairing/check-in sessions a week
- Definition of done: PR opened → tests pass → one review round → merged. No PR should outlive the sprint.

## Scope boundaries
Coding interns work on client UI, small server endpoints, and test coverage. Live production paths tied to multi-tenant core features are out of scope for interns.

## Who to contact
Your track coordinator (assigned at kickoff) for day-to-day blockers; escalate to the engineering lead for anything touching schema or auth.

## What "good" looks like at sprint end
At least one merged PR, and a short handoff note: what you did, what's unfinished, what you'd tell the next person.
