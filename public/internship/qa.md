# Project Zoe Internship — QA Testing Track

## What it is
Executing a documented set of test cases against the live Project Zoe platform, logging results and bugs clearly. Sprint length: 4-6 weeks. Low technical bar, high value — this keeps the platform trustworthy as more churches join.

## Entry bar
- Comfortable following a written checklist precisely, step by step
- Patient with repetitive, detail-heavy work
- No coding background required

## Trial task (this is the real interview)
Within 48 hours:
1. Execute 5 assigned test cases from the test plan
2. Log each result (pass/fail) using the bug-report template below

If your logs are clear and accurate, you're in.

## Bug report template
For every failure:
- **Steps taken**
- **Expected result**
- **Actual result**
- **Screenshot** (if visual)

## How work is assigned
- The existing Google Sheet test plan is broken into modules, one assigned per intern
- Results tracked in a shared sheet/GitHub issues so gaps are visible day to day, not just at sprint end
- Weekly short check-in with your track coordinator

## Bonus value
If you're also doing user training sessions (or hear from someone who is), UX issues they run into are useful QA input — flag them the same way as a normal bug.

## Who to contact
Your track coordinator for questions on any test case or the bug template.

## What "good" looks like at sprint end
All assigned modules executed, bugs logged clearly enough that an engineer can reproduce them without asking follow-up questions.
